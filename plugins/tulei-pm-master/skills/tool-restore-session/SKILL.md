# 项目记忆恢复 Skill

## 核心职责

确保 AI 助手始终拥有最新的项目背景、业务目标和任务进度，避免“失忆”导致重复劳动或方向偏差。

## 1. 触发判定 (Activation)

在以下情况下，AI 必须主动运行此 Skill 中的逻辑：

- 每次对话开始（Initial start）。
- 切换工作环境或设备后。
- 当 AI 发现无法确定 `task.md` 的当前状态时。

## 2. 深度恢复步骤 (Procedures)

### 第一步：读取核心规则

- **目标文件**：`project_context.md`
- **操作**：完整读取该文件，重点吸收 `## 2. 核心业务目标` 和 `## 3. 关键规则与约束`。

### 第二步：对齐任务进度

- **目标文件**：`task.md`
- **操作**：通过 `view_file` 确认哪些任务已完成 `[x]`，哪些在进行中 `[/]`，并标记当前最重要的 Top 1 任务。

### 第三步：理解长期规划

- **目标文件**：根据 `project_context.md` 中的索引，读取 `实施计划.md`。

## 3. 状态反馈 (Feedback)

完成读取后，AI 必须向用户进行一次“记忆同步报告”：

1. **状态汇总**：简洁描述项目目前在哪一阶段。
2. **任务确认**：确认正在处理的任务。
3. **就绪提问**：询问用户“是否继续 [具体任务]？”

## 4. 注意事项

- 严禁在未读取 `project_context.md` 的情况下做出重大技术决策。
- 如果发现 `task.md` 的内容与当前对话存在冲突，必须优先以 `task.md` 为准或询问用户。
