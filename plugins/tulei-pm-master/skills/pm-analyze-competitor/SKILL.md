# SKILL: 竞品商业模式分析

## 元信息

- **名称**: pm-competitor-analysis
- **版本**: 1.0
- **适用场景**: 基于APP截图或公开信息分析竞品商业模式
- **输出物**: 竞品商业模式分析报告（Markdown格式）
- **参考案例**: 金太阳项目竞品分析（103张截图，5个竞品）

---

## 触发条件

当用户需要以下任务时触发此SKILL：
- "分析这个竞品的商业模式"
- "帮我做竞品分析"
- "看看这个APP是怎么赚钱的"
- 用户提供了竞品APP截图

## 分析框架

### 1. 基础信息采集

```
每个竞品需要收集：
├── 产品定位（一句话描述）
├── 目标用户
├── 核心功能
├── 商业模式类型
├── 定价策略
└── 数据规模（用户量、收入等，如有）
```

### 2. 商业模式拆解

```
分析维度：
├── 流量入口（如何获客）
│   ├── 线上渠道（应用商店、搜索、社交）
│   ├── 线下渠道（实体产品、地推）
│   └── 品牌/口碑
│
├── 核心变现（如何赚钱）
│   ├── 订阅制（VIP会员）
│   ├── 单次购买（课程、内容）
│   ├── 服务收费（1对1、咨询）
│   ├── 硬件销售
│   └── 广告变现
│
├── 用户留存（如何提升粘性）
│   ├── 游戏化设计
│   ├── 社交功能
│   ├── 内容更新
│   └── 积分/会员体系
│
└── 成本结构
    ├── 内容成本
    ├── 技术成本
    ├── 运营成本
    └── 获客成本
```

### 3. 截图分析方法

当用户提供APP截图时，按以下维度分析：

| 截图类型 | 分析重点 |
|:--|:--|
| **首页/主界面** | 核心功能布局、用户路径设计 |
| **会员/VIP页** | 定价策略、权益设计、促销手段 |
| **功能页** | 免费vs付费功能差异 |
| **支付页** | 支付方式、价格锚点 |
| **个人中心** | 用户体系、积分/等级设计 |

## 常见商业模式类型

### 1. 免费增值（Freemium）
- **特点**：基础功能免费，高级功能付费
- **代表**：Spotify、Dropbox、多邻国
- **关键指标**：免费→付费转化率（通常1-5%）

### 2. 订阅制（Subscription）
- **特点**：按时间周期收费
- **代表**：Netflix、考霸刷题宝
- **关键指标**：续费率、LTV

### 3. 交易抽成（Marketplace）
- **特点**：平台撮合交易，收取佣金
- **代表**：淘宝、滴滴、知识星球
- **关键指标**：GMV、抽成比例

### 4. 内容付费（Pay-per-content）
- **特点**：单次购买内容
- **代表**：得到、喜马拉雅
- **关键指标**：客单价、复购率

### 5. 服务收费（Service）
- **特点**：提供人工服务收费
- **代表**：天星教育1对1
- **关键指标**：客单价、服务成本

## 注意事项

1. **数据来源标注**：明确标注数据来源和可信度
2. **避免主观臆断**：基于截图/事实分析，推测内容标注"推测"
3. **聚焦可借鉴点**：分析目的是为自己产品服务
4. **定价敏感**：VIP定价是核心参考，务必准确记录
