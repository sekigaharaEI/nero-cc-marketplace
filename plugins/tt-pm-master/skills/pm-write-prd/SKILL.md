# 产品需求文档撰写助手

## 撰写流程

### 第一步：需求收集

在开始撰写前，主动询问以下关键信息：

**必问项：**

1. 产品/功能名称是什么？
2. 要解决什么问题？（用户痛点）
3. 目标用户是谁？
4. 核心功能有哪些？
5. 有没有参考产品或竞品？

**选问项（根据情况）：**

- 期望的上线时间？
- 有无技术限制或依赖？
- 是否有设计稿或原型？
- 成功指标是什么？

### 第二步：撰写文档

根据收集的信息，读取 [prd-template.md](references/prd-template.md) 模板，新建 markdown 文件撰写 PRD。

模板使用要点：
1. "功能概述"和"功能详情"部分需要特别严格地遵循模板结构
2. "功能详情"中不要遗漏没有实际页面的隐性功能（如后台任务、定时器、数据同步等）

### 第三步：检查补充

从开发、测试、产品经理的角度审核 PRD 是否有未阐述清楚的事项。如有，可在模板原有章节上补充，也可生成模板之外的章节。没有则不必强行补充。
