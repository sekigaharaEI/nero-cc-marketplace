# 会话进度存档 Skill

## 1. 核心目标

解决“下次启动不知道从哪儿开始”的问题，确保项目状态、决策依据和进度被正确记录在 `00_Project_Context/project_context.md` 中。

## 2. 操作指南 (Procedures)

### 第一步：同步任务清单 (`00_Project_Context/task.md`)

- **原则**：诚实地检查哪些任务已彻底完成 `[x]`，哪些仍在进行中 `[/]`。
- **动作**：总结今天完成的任务数量及产出物。

### 第二步：更新核心上下文 (`00_Project_Context/project_context.md`)

- **目标位置**：跳转到 `## 5. 当前进度快照 (Current Status)`。
- **更新策略**：
  - **最新状态**：按时间顺序简述今天的里程碑。
  - **下一步指令**：这是给“未来的自己”留的便条。必须精准指出下一次开启会话时应该进行的动作。

### 第三步：提交保证

- 确认所有生成的文件都已按“中文命名规范”保存到正确的工作空间。

## 3. 注意事项 (Anti-patterns)

- **拒绝模糊总结**：禁止写“今天推进了项目进度”。必须具体到“完成了竞品分析 A 表，重构了 B 组件”。
- **单点故障检查**：检查是否有未完成的 `TODO` 悬而未决。
