# 目标：竞品 APP 深度商业分析 (带完整性检查)
**任务背景**：用户希望通过简单的截图文件夹，分析出竞品的核心商业逻辑。作为 AI，你不仅是分析师，还是“截图审核员”。如果关键页面缺失（导致无法判断能否赚钱），你必须暂停并要求用户补充。
**执行流程**：
1.  **初始化与目录扫描**
    -   获取截图根目录路径。
    -   使用 `list_dir` 扫描目录，浏览所有文件名。
2.  **完整性校验 (Critical Check)**
    -   **检查点 1：变现核心**
        -   寻找类似 `VIP`, `Price`, `Subscription`, `Paywall`, `Pro`, `会员`, `订阅`, `购买` 等关键词的文件或文件夹。
        -   如果没有找到：**立即暂停**。告诉用户：“我没有发现**付费界面/会员详情页**的截图。为了分析商业模式，请补充一张点击‘升级/购买’后的价格展示页截图，并放入该文件夹。补充好后请再次运行此指令。” -> **结束当前回合**。
    -   **检查点 2：核心功能**
        -   浏览首页 (`Home`) 和功能页。
        -   如果只有登录页 (`Login`) 或只有引导页 (`Splash`)，内容太少：**立即暂停**。告诉用户：“截图似乎主要集中在启动流程，缺少**核心功能界面**。请补充几张 APP 内部主界面的截图。” -> **结束当前回合**。
3.  **深度分析 (通过校验后)**
    -   如果以上关键检查点都通过（或者用户确认只有这些图），则开始深入阅读每一张关键截图。
    -   重点识别：
        -   **定价策略**：是订阅制 (SaaS) 还是单次买断？有无分级 (Silver/Gold)？
        -   **诱导机制**：哪些功能被打上了“锁”图标？
        -   **流量闭环**：是否有分享赢会员、邀请码等裂变设计？
4.  **生成分析报告**
    -   在截图根目录（或当前工作区）生成 `app_business_analysis.md`。
    -   **报告结构**：
        1.  **商业模式一句话总结**：(例：工具免费+高级功能订阅+去广告付费)
        2.  **变现漏斗分析**：
            -   入口：用户如何进来？
            -   卡点：在哪里被拦截要求付费？（截图中体现的 Paywall 位置）
            -   商品：卖的是“效率”、“特权”还是“内容”？
        3.  **核心资源/成本推演**：
            -   开发成本估算。
            -   运营维护成本。
        4.  **改进建议**：基于竞品截图，指出我们如果做，可以在哪里优化。
**交互原则**：
-   不要强行脑补。如果缺少关键信息，宁可多问一次用户，也不要出一份瞎编的报告。
