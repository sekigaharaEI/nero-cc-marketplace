# picture-set.md 构建规范

## 文件格式

picture-set.md 是一个 Markdown 文件，每张图片的描述用 `---` 分隔。

## 文件命名

- 默认命名：`picture-set.md`
- 如果当前目录已存在同名文件，加时间戳后缀：`picture-set-20260210-1430.md`

## 文件头

所有 picture-set.md 文件以以下标题开头：

```markdown
# 图片集描述

---
```

## 文生图条目格式

```markdown
【第N张】
风格：{风格关键词}
内容：{详细的图片内容描述}
```

### 描述扩写规则

如果用户的描述过于简短（少于30字），Claude **必须**主动扩写：

1. 补充具体的视觉元素（颜色、光影、构图）
2. 补充环境细节（背景、氛围、天气）
3. 补充风格特征（画风、色调、质感）
4. 扩写后的描述应在 50-150 字之间
5. **扩写后必须展示给用户确认**

### 风格关键词参考

如果用户未指定风格，根据内容推荐：

| 内容类型 | 推荐风格 |
|----------|----------|
| 人物/肖像 | 数字绘画 / 油画 |
| 风景/自然 | 摄影 / 水彩 |
| 科技/未来 | 赛博朋克 / 科幻概念艺术 |
| 食物 | 美食摄影 |
| 抽象概念 | 抽象艺术 / 超现实主义 |
| 可爱/卡通 | 插画 / 动漫风格 |
| 商业/产品 | 商业摄影 / 3D渲染 |
| 中国风 | 国潮插画 / 工笔画 |

### 描述语言

不做强制要求，用户用中文或英文描述均可，Claude 保持用户的原始语言。

## 改图条目格式

```markdown
【第N张】

**参照原图**

"原图文件名（不含扩展名）"

**修改意见**

{用户的修改意见}
```

### 原图文件说明

- 原图文件名用于 picture-set.md 中引用，**不含扩展名**
- NotebookLM 会对上传的图片进行 OCR 识别，picture-set.md 中的文件名引用会让 NotebookLM 将描述与对应图片关联
- 如果原图文件名含中文或特殊字符，保持原样

## 混合模式

文生图和改图条目可以混合出现在同一个 picture-set.md 中，按用户指定的顺序排列，序号连续。

## 完整示例

```markdown
# 图片集描述

---

【第1张】
风格：赛博朋克
内容：霓虹灯照耀下的雨夜街道，高耸入云的摩天大楼表面布满全息广告屏。画面中央一只精密的机械手臂与人类的手正在握手，机械手臂呈半透明状态，内部可见蓝色电路光芒和精密齿轮。天空中飞行着无人机和飞行汽车留下光轨，地面湿漉漉的倒映着紫色、蓝色、粉色交织的灯光。

---

【第2张】

**参照原图**

"原图_第2张"

**修改意见**

把背景改成星空，添加流星划过天际

---

【第3张】
风格：水彩
内容：春天的樱花树下，一只橘猫蜷缩在木质长椅上睡觉。花瓣随风飘落，阳光透过树冠洒下斑驳光影。远处是一条蜿蜒的石板小路，两旁种满了紫色的鸢尾花。整体色调温暖柔和，营造宁静惬意的午后氛围。

---
```

## 数量限制

- 最多 20 张（NotebookLM 幻灯片页数上限）
- 如果用户需要超过 20 张，提示分批生成
