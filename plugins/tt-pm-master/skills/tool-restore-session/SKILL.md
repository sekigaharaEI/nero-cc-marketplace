# 恢复项目上下文

默认把恢复理解为“读锚点文件并恢复长期记忆 + 当前状态”，不是复述全部历史。

## 先读

1. `00_Project_Context/alignment.md`
2. `00_Project_Context/project_context.md`
3. `00_Project_Context/迭代记录.md`
4. `00_Project_Context/next.md`
5. `00_Project_Context/task.md`

只有在需要理解阶段方案时，才补读：

- `00_Project_Context/实施计划.md`

如果需要继续具体写作，再看最新一周的：

- `终稿`
- `效果验证`
- `投稿复盘`

## 恢复目标

恢复后必须能回答：

- 当前唯一目标是什么
- 项目背景和长期偏好是什么
- 当前正在做什么
- 已完成了什么、没完成什么
- 当前这轮到底在验证什么
- 已达成了哪些观点和共识
- 现在最优先的下一步是什么
- 是否需要继续最新一周样稿

## 输出格式

恢复完成后，向用户给出 4 点：

1. 当前阶段一句话总结
2. 当前正在做什么 + 已完成/未完成的关键项
3. 当前共识和偏好
4. 建议直接继续的下一步

如果信息已经足够，不要再追问泛问题。

## 恢复规则

- 优先以 `alignment.md` 为当前共识
- 优先以 `project_context.md` 为背景和偏好
- 优先以 `next.md` 为当前动作
- 优先以 `迭代记录.md` 为最近变化
- 优先以 `task.md` 为已完成/未完成状态
- `实施计划.md` 默认不是必读文件
- 如果恢复后仍答不出上面的任一问题，明确指出缺的是哪一类信息
- 只有在锚点文件互相冲突时，才指出冲突并请用户拍板

## 典型触发语

- `恢复会话`
- `继续上次`
- `先同步一下状态`
- `我们现在进行到哪了`
