# Teacher Tui产品经理大师 - 帮助文档

请以表格形式向用户展示本插件包含的所有能力：

## 产品经理核心能力

| 能力名称 | 触发方式 | 功能描述 | 使用场景 |
|:---------|:---------|:---------|:---------|
| **PRD撰写** | "写PRD"、"写需求文档"、"帮我写一个需求" | 专业的产品需求文档撰写助手，包含完整的PRD模板、功能架构图、状态机图、交互细节等 | 新功能开发、产品迭代、需求评审前准备 |
| **竞品分析** | "分析竞品"、"帮我做竞品分析"、提供竞品APP截图 | 基于APP截图或公开信息，深度拆解竞品商业模式、定价策略、变现路径 | 立项调研、商业模式设计、差异化定位 |
| **APP分析** | 提供APP截图目录 | 根据APP截图反向分析产品功能、商业模式及资源投入，自动检查截图完整性 | 快速了解一款APP的商业逻辑、学习竞品设计 |
| **商业模式规划** | "帮我做商业模式规划"、"设计盈利模式"、"写商业计划书" | 生成面向高管汇报的商业模式规划方案，包含财务预测、风险分析、决策建议 | 新产品立项、投资汇报、战略规划 |
| **产品评审** | 提供产品方案文档或截图 | 模拟"产品评审委员会"（CXO/CTO/COO/CPO四方会审），输出犀利的评审报告 | 方案自检、评审前预演、发现产品漏洞 |
| **评审意见处理** | 收到评审意见后使用 | 作为产品经理独立思考，对每条评审意见给出采纳/部分采纳/拒绝的答复及理由 | 评审后的意见处理、跨部门沟通、决策记录 |

## 工具类能力

| 能力名称 | 触发方式 | 功能描述 | 使用场景 |
|:---------|:---------|:---------|:---------|
| **会话存档** | 结束工作时、完成阶段性任务时 | 将当前会话进度、任务状态、决策依据写入持久化文档 | 下班前存档、阶段性总结、交接工作 |
| **记忆恢复** | 新会话开始时、切换设备后 | 读取项目上下文、任务清单、实施计划，恢复工作状态 | 新一天开始工作、接续之前的任务 |
| **技能迁移** | "迁移skills到workflows" | 将 `.claude/skills` 批量迁移转换为 `.agent/workflows` 格式 | 技能格式升级、跨工具迁移 |
| **NotebookLM** | `/notebooklm`、"生成播客"、"创建笔记本" | 完整的 Google NotebookLM API 支持，创建笔记本、添加素材、生成播客/视频/幻灯片等 | 内容创作、知识整理、播客生成 |
| **文档转幻灯片** | `/doc-to-slides`、"文档转PPT"、"生成幻灯片" | 将本地文档上传到 NotebookLM 并自动生成幻灯片 PDF | 快速制作演示文档、汇报材料 |
| **长文本写入** | 写入超过3000字的内容时自动触发 | 分块写入超长文本，避免单次输出 token 限制 | 创建长文档、生成大量内容 |

## 使用建议

1. **新项目启动**：先用「竞品分析」了解市场，再用「商业模式规划」设计盈利模式
2. **需求开发**：用「PRD撰写」输出规范文档，用「产品评审」自检方案
3. **日常工作**：开始时用「记忆恢复」，结束时用「会话存档」
4. **评审流程**：收到意见后用「评审意见处理」，保持专业且有理有据的回复
